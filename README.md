# 台灣銀行交易資料清洗管線 (Banking Data Cleaning Pipeline)
針對台灣銀行場景設計的交易資料清洗管線，涵蓋診斷 → 清理 → 驗證完整流程，含日誌記錄與命令列參數，可作為排程腳本無人執行。

## 資料的七類髒法
本專案使用合成資料，刻意埋入台灣金融資料實務中常見的髒污：
- **民國年與西元年混雜：** 113/03/11 與 2024/3/11 同欄並存，需 +1911 轉換
- **五種日期格式：** 西元斜線、民國斜線、ISO 橫線、歐式 DD-MM-YYYY、中文「年月日」
- **前導零遺失風險：** 客戶 ID(7碼)與分行代碼(3碼)以 dtype=str 讀入，防止被當成數字吃掉開頭的零
- **全形字元：** 全形數字 ８００ 混入金額欄，以 NFKC 正規化統一
- **空值的多重偽裝：** NaN、N/A、None、-、空字串、空格六種型態
- **疊加式髒金額：** 千分位逗號、NT$ 前綴、前後空白可同時出現在同一筆
- **重複資料：** 完全重複列(技術事故)與業務重複(需人工裁決)，分開處置

## 設計思路
- **診斷先於清理：** 先用格式骨架分析(將數字替換為佔位符)盤點所有格式變體，再動手清理，避免「只清看得見的髒」。
- **分流 vs 管線：** 日期的五種格式互斥、處理方式互相衝突，採遮罩分流各自解析；金額的髒法可疊加、清潔動作互不干擾，採單一管線串接處理。
- **原始欄位唯讀：** 所有清理結果寫入 _clean 新欄位，原始資料保留作為對照與驗證依據。

## 驗證設計
- **通則檢查(assert)：** 對任何一批資料都成立的規則——金額為正、類型白名單、識別碼格式、無未來日期、無完全重複。違反即中止。
- **快照監控(logging)：** 批次特定的事實——列數、空值數、業務重複筆數——記入日誌供跨批次趨勢追蹤，不阻擋流程。
- **金額驗證：** 採獨立雙路徑對帳，清洗管線與驗證路徑使用不同邏輯計算總額，兩者一致才視為通過。

## 執行方式

##### pip install pandas
##### python clean_transactions.py   #使用預設路徑
##### python clean_transactions.py --input transactions_0711.csv --output clean_0711.csv   #指定輸入輸出
